## 混合モデル

より高性能なモデルで複雑な機能を計画し、高速なモデルで効率的に実行—すべて同じセッション内で再起動することなく実現できます。

`/model` コマンドにアクセスして、実行用とは別にスペックモデルを設定できます。アーキテクチャ計画にはSonnet 4.5を、実装にはHaiku 4.5を使用したり、さまざまな組み合わせをテストしてコストとレイテンシを最適化できます。

<Note>詳しくは[混合モデル](/jp/cli/configuration/mixed-models)を参照してください。</Note>

## カスタムサブエージェント

ワークフロー専用のDroidを構築できます。ツール、モデル、動作を設定しましょう。

**動作の仕組み**

1. `/settings` でCustom Droidsを有効にする
2. `/droids` を起動してDroids Managerを開く
3. サブエージェントを手動で作成するか、Factoryに生成させる
4. 各サブエージェントが使用するツールとモデルを設定する
5. セッション中にカスタムDroidsにタスクを委任する

**用途例**

- コードレビューとコンプライアンスチェック用のセキュリティスペシャリスト
- アーキテクチャ分析用のコードベースエクスプローラー
- 包括的なテストカバレッジ用のテストジェネレーター
- APIと機能ドキュメント用のドキュメンテーションライター

<Note>[カスタムDroid](/jp/cli/configuration/custom-droids)を見る</Note>

## PRレビュー用Factory GitHub App

プルリクエスト内で直接コードレビューを自動化します。FactoryのGitHub Appは、手動でのCLI設定を必要とせず、完全なコンテキストでインラインフィードバックを提供します。

[Install Factory Droid Review on GitHub Marketplace →](https://github.com/marketplace/actions/factory-droid-review)

リポジトリにアプリを追加し、`Factory-AI/droid-code-review@latest` アクションでワークフローを設定します。DroidsがPRの変更を分析し、特定の行に詳細なコメントを残すことで、問題を早期に発見し、コード品質基準を維持します。

**例**

```yaml
- name: Droid Review
  uses: Factory-AI/droid-code-review@latest
```
