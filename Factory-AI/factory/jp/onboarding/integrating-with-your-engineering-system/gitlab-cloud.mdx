このガイドでは、Factory を GitLab Cloud と統合し、Factory プラットフォームが GitLab リポジトリとシームレスに連携できるようにする手順を説明します。

## 前提条件

- 管理者権限を持つ Factory アカウント
- GitLab Cloud 組織への管理者アクセス権

## 統合手順

<Steps>
  <Step title="Factory プラットフォームにアクセス">
    Factory アカウントにログインし、Settings の Integrations セクションに移動します。
  </Step>
  <Step title="GitLab 統合を開始">
    GitLab の横にある「Connect」をクリックして統合プロセスを開始します。「Manage GitLab Permissions」を選択します。
  </Step>
  <Step title="Factory の GitLab Application を認可">
    GitLab にリダイレクトされます。要求される権限を確認し、「Authorize」をクリックして、Factory の安全な OAuth2 GitLab Application が組織へアクセスすることを許可します。
  </Step>
  <Step title="権限を確認して承認">
    Factory が最適に機能するには、主に 2 つの OAuth scope が必要です。

    - **api**: この包括的な scope により、Factory はリポジトリの読み書き、issue と merge request の管理、commit へのコメントを行えます。
    - **read_user**: この scope により、Factory はユーザー情報へアクセスでき、さまざまな操作でより適切なコンテキストを得られます。

    続行する前に、これらの scope が選択されていることを確認してください。
  </Step>
  <Step title="リポジトリアクセスを設定">
    Factory にアクセスさせるリポジトリを選択します。認可済みの group または personal namespace 内にある、現在および将来のすべてのリポジトリへのアクセスを許可することもできます。
  </Step>
  <Step title="統合を確認">
    権限を付与すると Factory に戻ります。統合ステータスが「Connected」と表示されていることを確認してください。
  </Step>
</Steps>

## 検証

統合が正しく動作していることを確認するには、次の手順を実施します。

1. テスト用リポジトリを作成するか、機密データが最小限の既存リポジトリを使用します。
2. merge request の作成など、簡単なワークフローを開始します。
3. Factory が想定どおりにリポジトリへアクセスし、操作できることを確認します。

## ベストプラクティス

- Factory に付与した権限を定期的に見直し、監査する。
- GitLab の group レベル設定を使って、複数プロジェクトへのアクセスを効率的に管理する。
- GitLab 組織のセキュリティ設定を常に最新に保つ。
- GitLab の設定にある認可済みアプリケーション一覧を定期的に見直す。

## トラブルシューティング

統合時に問題が発生した場合は、次を確認してください。

- Factory と GitLab の両方で必要な管理者権限があること。
- 権限関連の問題がないか GitLab の監査ログを確認すること。
- OAuth application が GitLab 設定内で正しく認可されていること。

問題が解決しない場合は、具体的なエラーメッセージやスクリーンショットを添えて Factory サポートにお問い合わせください。
