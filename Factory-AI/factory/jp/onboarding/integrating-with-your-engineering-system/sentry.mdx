このガイドでは、Factory を Sentry と統合し、Factory プラットフォームがエラートラッキングデータにアクセスして活用できるようにする手順を説明します。

## 前提条件

- 管理者権限を持つ Factory アカウント
- Sentry 組織への管理者アクセス権
- アプリケーション用の Sentry プロジェクトが設定済みであること

## 統合手順

<Steps>
  <Step title="Factory プラットフォームにアクセス">
    Factory アカウントにログインし、Settings の Integrations セクションに移動します。
  </Step>
  <Step title="Sentry 統合を開始">
    Sentry の横にある「Connect」をクリックして統合プロセスを開始します。
  </Step>
  <Step title="Factory の Sentry 統合をインストール">
    Sentry にリダイレクトされます。要求内容を確認し、組織内で Factory の Sentry 統合をインストールすることを承認します。
  </Step>
  <Step title="アクセス権限を設定">
    Factory が最適に機能するには、次の権限が必要です。

    - **Issue & Event Access**（読み取り権限）: issue の詳細、stack trace、関連 event を取得できます
    - **Organization Access**（読み取り権限）: 組織全体の設定や metadata にアクセスできます
    - **Project Access**（読み取り権限）: プロジェクト固有の情報や設定にアクセスできます
  </Step>
  <Step title="統合を確認">
    権限を付与すると Factory に戻ります。統合ステータスが「Connected」と表示されていることを確認してください。
  </Step>
</Steps>

## 使用方法

統合後、Factory では次のことが可能になります。

1. **issue 詳細へのアクセス**: stack trace、breadcrumb、コンテキストデータを含む包括的なエラー情報を表示
2. **issue 検索**: Sentry プロジェクト全体から特定の issue を検索
3. **エラーパターンの分析**: エラー傾向や一般的な障害ポイントを把握
4. **コンテキスト統合**: コード変更の議論やデバッグ時に Sentry データをコンテキストとして使用

### issue コンテキスト

Factory のチャットで Sentry issue URL を共有すると、アシスタントは自動的に次を実行します。

1. issue の詳細を取得
2. 関連する stack trace を抽出
3. エラーコンテキストと breadcrumb を含める
4. エラー情報に基づくインサイトを提供

Sentry issue URL の例:

```
https://sentry.io/organizations/your-org/issues/12345/
```

## ベストプラクティス

- Factory に付与した権限を定期的に見直し、監査する
- Sentry のプロジェクトレベル設定を使ってアクセス制御を細かく調整する
- Sentry 組織のセキュリティ設定を常に最新に保つ
- 統合の利用状況と効果を定期的に見直す

## トラブルシューティング

統合時に問題が発生した場合は、次を確認してください。

- Factory と Sentry の両方で必要な管理者権限があること
- 権限関連の問題がないか Sentry の監査ログを確認すること
- 組織に競合するサードパーティアプリケーション制限がないこと

問題が解決しない場合は、具体的なエラーメッセージやスクリーンショットを添えて Factory サポートにお問い合わせください。
